# 홈트 뽑기

홈트용 서킷 운동 랜덤 생성기 + 음성 가이드 타이머 PWA.

- 하체·푸시·풀·코어·전신 **5패턴에서 하나씩 뽑아** 매번 다른 서킷 생성
- 운동 중 **음성이 다음 동작과 폼 포인트를 읽어줘서** 폰을 보지 않아도 됨
- 타이머 중 화면 꺼짐 방지(Wake Lock), 완료 기록·연속 일수 스트릭
- 설치 없이 홈 화면에 추가해 앱처럼 사용, 오프라인 동작

> 이 앱은 **"회사원도 이런 앱을 만들 수 있다"**를 보여주는 교육용 데모 시리즈의 원형입니다.
> [claude.ai](https://claude.ai) 대화창에서 아티팩트로 시작해, 하루 만에 배포·공유·문의까지 갖춘
> 제품이 됐습니다. 같은 틀에서 [점심 뽑기](https://itda-demo-lunch-pick.vercel.app),
> [고장 길잡이](https://itda-demo-fix-guide.vercel.app) 같은 시리즈가 이어지고 있습니다.

## 🙋 이 앱, 나도 만들 수 있을까?

**네, 코딩을 몰라도 됩니다.** 이 앱은 사람이 코드를 타이핑한 것이 아니라 AI(Claude)에게 **말로 시켜서** 만든 것입니다. 실제 제작 순서도 참고가 됩니다: ① claude.ai 대화창에서 아티팩트(작동하는 시안)로 시작 → ② 마음에 들자 Claude Code로 옮겨 진짜 앱으로 다듬기 → ③ 무료 배포.

### 준비물

1. **Claude** — [claude.ai](https://claude.ai) (웹) 또는 Claude Code(터미널). 처음이라면 claude.ai 아티팩트만으로 충분합니다.
2. **만들고 싶은 것에 대한 구체적인 설명** — 아래 프롬프트처럼.

### 따라 만들기 — 이 프롬프트를 그대로 써보세요

```
홈트레이닝 서킷 운동 앱을 만들어줘.

- 운동을 하체/푸시/풀/코어/전신 5개 패턴으로 분류하고,
  "뽑기" 버튼을 누르면 각 패턴에서 하나씩 랜덤으로 뽑아
  오늘의 5동작 서킷을 만든다
- 시작하면 타이머가 준비 10초 → [운동 30초 → 휴식 15초] 반복으로 진행되고,
  휴식 중에는 다음 동작 이름을 크게 예고해준다
- 라운드 수·운동/휴식 시간은 설정 가능
- 운동 목록에 내가 아는 동작을 추가할 수 있다
- 완료하면 기록이 남고 연속 일수를 보여준다
- 데이터는 서버 없이 브라우저에만 저장
- 어두운 배경에 큰 숫자, 모바일 우선 디자인
```

한 번에 완성되지 않아도 괜찮습니다. **"타이머 숫자를 더 크게"**, **"휴식 때 다음 동작을 미리 읽어줘"** 처럼 대화로 고쳐나가는 것이 정상 과정입니다 — 이 앱도 그렇게 커졌습니다.

**음성 가이드의 비밀**: 앱이 말을 하는 건 실행 중에 AI를 부르는 게 아니라, 나올 수 있는 문장을 **만들 때 미리 음성 파일로 생성**해 둔 것입니다(ElevenLabs). 그래서 API 키도 사용료도 없이 오프라인에서 동작합니다. 운동 삽화도 같은 방식으로 AI 이미지 생성을 미리 돌려 담았습니다.

### 더 쉬운 방법 — 이 저장소를 통째로 보여주기

Claude Code에서 이 저장소를 열고:

```
이 앱과 같은 구조·디자인으로 "점심 뽑기" 앱을 만들어줘.
팀 점심 메뉴를 카테고리별로 뽑고, 최근 먹은 건 제외해줘.
```

실제로 이 프롬프트 한 줄 수준의 요청에서 [점심 뽑기](https://github.com/itda-demo-apps/demo-lunch-pick)가 나왔습니다.

## 📚 알아두면 좋은 개념 (비개발자용 최소 지식)

| 개념 | 한 줄 설명 |
|---|---|
| **아티팩트** | claude.ai 대화창 안에서 바로 실행되는 작은 앱 — 시안 만들기에 최적 |
| **웹앱** | 설치 없이 브라우저 주소로 여는 앱. 앱스토어 심사 없이 URL로 배포 |
| **PWA** | "홈 화면에 추가"하면 진짜 앱처럼 아이콘·전체화면·오프라인 실행이 되는 웹앱 |
| **localStorage** | 서버 없이 **내 기기 안에** 데이터를 저장하는 방식 |
| **빌드타임 AI** | AI를 실행 중에 부르지 않고 **만들 때만** 써서 음성·이미지·데이터를 미리 생성 |
| **배포** | 내 컴퓨터의 앱을 인터넷 주소로 여는 것. 이 시리즈는 Vercel(무료)을 씁니다 |

## 🔍 더 알고 싶을 때 조사하는 법

1. **Claude에게 바로 물어보기** — "PWA가 뭐야? 초등학생에게 설명하듯"처럼 눈높이를 지정하세요.
2. **막히면 에러를 그대로 붙여넣기** — 해석하려 하지 말고 복사해서 "이거 왜 이래?"
3. **공식 문서는 AI와 함께 읽기** — URL을 주고 "요약해줘, 내 상황엔 뭘 쓰면 돼?"

## 개발자용: 직접 실행

```bash
npm install
npm run dev        # http://localhost:5173 — 같은 와이파이 폰에서도 접속 가능
npm run build      # dist/ + 서비스워커 생성
npm run preview    # 빌드 결과 확인
npm run voice      # ElevenLabs 음성 클립 재생성 — .env의 ELEVENLABS_API_KEY 필요
```

### iPhone 홈 화면에 추가

1. HTTPS로 배포된 주소를 iPhone Safari로 연다
2. 공유 버튼 → **홈 화면에 추가**
3. 홈 화면 아이콘으로 실행하면 주소창 없는 전체 화면 앱으로 열린다 (오프라인도 동작)

자세한 구조·규약·로드맵은 [CLAUDE.md](./CLAUDE.md), 기능 스펙은 [docs/기획.md](./docs/기획.md) 참고.
